Введение

          Платформа SAP Solution Manager предоставляет функциональные возможности в области внедрения, эксплуатации и постоянной оптимизации используемого решения mySAP Business Suite. Эта система предоставляет центральную точку доступа к методам инструментальных средств и предварительно сконфигурированному содержимому, которыми можно воспользоваться в ходе внедрения, тестирования и продуктивной эксплуатации решений.

        SAP Solution Manager предназначен для управления всем ландшафтом решения mySAP Business Suite клиента, включая взаимодействие с внешними (не SAP) системами, т.е. SAP Solution Manager функционирует в рамках используемой в компании ИТ-среды.

        SAP Solution Manager является дополнением, находящимся над SAP Web Application Server, mySAP Workplace 2.11 или SAP R/3 4.6C.

 1.1 Основные возможности SAP Solution Manager

       Платформа SAP Solution Manager основана на стратегии компании SAP в области управления ИТ-сервисами и приложениями и позволяет наиболее эффективно использовать сервисы поддержки SAP.

 SAP Solution Manager обеспечивает (см. рис. 1.1):

   

Рисунок 1.1 – Возможности SAP Solution Manager.

  • Реализацию проектов внедрения и глобального развертывания решений mySAP Business Suite с процессно-ориентированным подходом к конфигурированию и тестированию. SAP Solution Manager может использоваться на всех фазах внедрения методологии ASAP; 

  • SAP Solution Manager предоставляет единую точку доступа к системам компонентов для работ по созданию дизайна, конфигурированию и тестированию;

  • Доступ к лучшим методам для mySAP Business Suite, маршрутным картам, шаблонам документов, предварительно сконфигурированным бизнес-процессам.

  • Мониторинг и оптимизацию всего комплекса решений SAP;

  • Повышение надежности функционирования системной среды за счет профилактического контроля решений, а также за счет настройки синхронизации и проверок непротиворечивости неоднородных сред;

  • Проективное распознавание потенциальных проблем благодаря мониторингу и системе отчетов.

  • Доступ к централизованной справочной платформе для поддержки пользователей SAP, создание внутренней системы технической поддержки.

  1.2 Структура SAP Solution Manager

Компоненты, входящие в платформу SAP Solution Manager разобьем на три группы, которые имеют следующие названия (см. рис. 1.2):

  • Интерфейс с компанией SAP

  • Информационное наполнениее

  • Инструментарий

   

Рисунок 1.2 – Обзор компонентов SAP Solution Manager.

Интерфейс с компанией SAP

        В процессе работы, чтобы обеспечить надежную бесперебойную эксплуатацию решений SAP, клиентам и партнерам SAP может потребоваться углубленная информация о решениях, техническая поддержка программного обеспечения или консультационные услуги. Клиентам требуется полная и актуальная информация о предлагаемых компанией SAP услугах в сфере обучения. Разработчикам и администраторам на предприятиях необходима своевременная и надежная техническая поддержка. SAP Solution Manager обеспечивает удобный интерфейс для взаимодействия с компанией SAP.

    Взаимодействие с платформой SAP Service Marketplace

Единая централизованная комплексная платформа сервиса и поддержки SAP Service Marketplace обеспечивает основу для коммуникации и сотрудничества между клиентами, партнерами и компанией SAP.

Информационные ресурсы SAP Service Marketplace разделены на несколько отдельных порталов: SAP Support Portal (Портал поддержки), SAP Developer Network (Портал для разработчиков), SAP Partner Portal (Портал для партнеров), SAP SMB Portal (Портал для компаний среднего и малого бизнеса), SAP Help Portal (Портал справочной службы) и SAP Community (Портал SAP-сообщества).

        Центральный портал находится по адресу http://service.sap.com.

        Для работы с этими информационными ресурсами с помощью Solution Manager можно создать необходимое число пользователей портала, настроить им права доступа и осуществлять администрирование этих настроек.

        Платформа SAP Service Marketplace включает в себя функциональные возможности системы обслуживания клиентов и содержит широкий спектр информации о компании SAP, ее продуктах и услугах. В портале представлены разнообразные тематические разделы, предназначенные для клиентов и партнеров (см. рис. 1.3):

   

Рисунок 1.3 – Информационные разделы в SAP Service Marketplace.

  • Карты решений SAP Solution Maps;

  • Подробная информация о решениях и технологиях, сведения о партнерах SAP и сертифицированных продуктах других производителей (дополнительное программное обеспечение);

  • Подробная информация о версиях для различных платформ, включая планируемые и существующие версии;

  • Информация о дополнительных разработках, учитывающих специфику стран СНГ;

  • Подробная информация о семинарах, проводимых в московском Сервисном центре SAP, а также в Сервисных центрах, расположенных в других странах;

  • Руководства по инсталляции для текущих версий решений SAP;

  • Новости, обзор событий, истории успешных проектов внедрения решений SAP;

  • Множество другой полезной информации.

С помощью платформы SAP Service Marketplace осуществляется обращение клиента SAP к информационному пакету SAP Best Practices (Лучшие практики SAP), который  включает методологии, набор документации и предварительно сконфигурированные примеры для комплекса решений mySAP Business Suite. 

    Дистанционные профилактические услуги

В SAP существуют специальные профилактические и предупредительные сервисыы, которые увеличивают безопасность используемого решения в течение всего его жизненного цикла. Помимо их локального использования клиент может воспользоваться услугой дистанционного запуска сервисов компанией SAP.

    Сообщения поддержки SAP

Если при внедрении или эксплуатации решения возникает проблема, связанная с работой системы, клиенту необходим совет эксперта. SAP Solution Manager открывает доступ к ряду инструментов при помощи которых можно создать сообщение для внутренней группы поддержки, получить доступ к локальной базе знаний SAP Notes, которые помогают решить проблему самостоятельно. Если клиент не может найти решение самостоятельно, SAP Solution Manager позволяет отправить созданное сообщение в подразделение SAP Active Global Support.

При получении сообщения о сбое в системе компания SAP снабжает клиента информацией относительно того, как обойти, избежать или исправить ошибки. Пользователи могут посылать сообщения в SAP круглосуточно в любой день недели. Клиенты из стран СНГ и Балтии могут выставлять сообщения на русском языке, но рекомендуется, по возможности, в качестве языка коммуникации использовать английский язык.

            Более подробно об обработке сообщений поддержки  мы поговорим в разделе "Служба поддержки".

Информационное наполнение

    Бизнес-сценарии

Решение SAP Solution Manager предоставляет экспертные знания в процессе прохождения всех фаз проекта внедрения. В состав SAP Solution Manager входит значительный объем бизнес-сценариев для внедрения комплекса решений mySAP Business Suite. Эти сценарии содержат предварительно определенные бизнес-процессы из Репозитария бизнес-процессов (RBP).

Поставляемый вместе с SAP Solution Manager Репозитарий бизнес-процессов представляет собой базу содержимого внедрения с предварительно определенными и многократно используемыми данными по бизнес-процессам. Репозитарий бизнес-процессов содержит предварительно сконфигурированные процессы, их графическое представление, поэтапные руководства по их внедрению, данные предварительной конфигурации (транзакции конфигурации, IMG-проект), а также материалы для тестирования (см. рис 1.4).

   

 Рисунок 1.4 – Бизнес-содержимое SAP Solution Manager.

    При создании концептуального проекта заказчики выбирают предварительно сконфигурированные бизнес-процессы, которые требуется внедрить, а затем адаптируют их к существующим требованиям на предприятии. Кроме того, клиент может сам создавать бизнес-процессы. В зависимости от бизнес-сценария, указанного в документе объема, будет доступно соответствующее содержимое из репозитария бизнес-процессов.

            Последнюю версию содержимого для Репозитария бизнес-процессов можно выгрузить с SAP Service Marketplace по адресу http://service.sap.com/solutionmanager.

При использовании бизнес-содержимого SAP информация о конфигурации и предварительной конфигурации уже считается присвоенной необходимым элементам системы. Следовательно, использование бизнес-содержимого SAP помогает снизить затраты на проектирование бизнес-процессов, а также соответствующую настройку, тестирование и обучение, выполняемые на более поздних этапах проекта.

Взаимодействие SAP Solution Manager и ARIS

При внедрении решений mySAP Business Suite наиболее эффективным считается процессно-ориентированный подход. Он может быть реализован за счет совместного использования системы SAP Solution Manager фирмы SAP и инструментальной среды ARIS for mySAP.com фирмы IDS Scheer AG. Это предоставляет следующие возможности:

  • Визуального отображения результатов проекта по внедрению системы на любой стадии его выполнения;

  • Добавления или удаления компонент системы непосредственно на графических моделях;

  • Запуска транзакций SAP для рассматриваемых бизнес-процессов (транзакции запускаются с сервера системы SAP);

  • Использование сценариев бизнеса (business scenarios) и ссылочных моделей R/3 (R/3 reference model) в ARIS for mySAP.com позволяет предприятиям настроить стандартные сценарии mySAP Business Suite на собственные нужды и документировать бизнес-процессы;

  • Дополнительная компонента ARIS™ Connectivity for SAP HR позволяет использовать данные об организационных единицах из SAP R/3 HR.

        Используя инструментальное средство ARIS for mySAP.com можно описать необходимые интерфейсы с системой SAP R/3, основываясь на бизнес-процессах предприятия.

        Для моделирования бизнес-процессов существует возможность экспорта модели процессов SAP R/3 в инструментальное средство моделирования ARIS for mySAP.com и последующей настройки на основе обработанных моделей IMG-проекта через SAP Solution Manager.

        Изменения, сделанные в референтной модели SAP, могут быть перенесены как из SAP в ARIS, так и обратно из ARIS в SAP.

Таким образом, ARIS for mySAP.com является функциональным и идеологическим расширением системы SAP Solution Manager для управления проектом по внедрению mySAP Business Suite.

 

Маршрутные карты

 Основным компонентом маршрутных карт являются ссылки на акселераторы и инструментальные средства, предназначенные для выполнения задач проекта. Актуальные маршрутные карты поставляются вместе с SAP Solution Manager. Они содержат стандартную методологию внедрения SAP и информацию о важнейших аспектах и фазах внедрения SAP.

В Solution Manager в работе с маршрутными картами имеются следующие возможности:

  • Осуществление навигации в структуре;

  • Фильтрация структурных элементов по ролям или тематическим областям;

  • Просмотр и присвоение документов;

  • Печать отдельных документов или структур с присвоенными документами;

  • Поиск определенных элементов в структуре и акселераторах;

  • Контроль статуса и создание примечаний для каждого структурного элемента, например, для пакета работ;

  • Создание сообщений.

        В SAP Solution Manager доступны следующие виды маршрутных карт (см. рис. 1.5).

   

 Рисунок 1.5 – Виды маршрутных карт.

        Маршрутная карта внедрения описывает методы организации и реализации проекта внедрения с функциональной точки зрения. Она содержит методологические предпосылки для выполнения задач по внедрению решения SAP. Сюда входят, помимо прочего, сведения о руководстве проектом и конфигурации бизнес-процессов, а также технические аспекты, процедуры тестирования и концепции обучения.

      Маршрутная карта внедрения дополняет маршрутную карту управления решением. Они построены на основе концепции вех с указанием критических точек синхронизации.    

        Глобальный шаблон определяется в сценариях глобального развертывания в головной фирме. В нем размешаются сценарии, документы концептуального проекта и конфигурации. Можно определить, какие компоненты шаблона релевантны для дочерних компаний, а также то, могут ли эти дочерние компании изменять шаблон в проектах развертывания, и в какой степени. Маршрутная карта глобального шаблона не синхронизирована с маршрутной картой управления решением.    

Инструментарий

        Эту группу компонентов мы рассмотрим с точки зрения их функционального назначения. Более подробно об инструментах SAP Solution Manager будет рассказываться в дальнейших главах.

Организация и ведение проекта внедрения решения SAP

            SAP Solution Manager представляет собой инструмент создания и ведения проекта по внедрению. В главе "Внедрение решения mySAP Business Suite" мы рассмотрим использование SAP Solution Manager  в фазах методологии ASAP, а пока несколько слов о возможностях платформы.

На этапе "Подготовка проекта" осуществляется централизованное определение всех задействованных систем и объема их использования в рамках проекта внедрения бизнес-решения, т.е. определяется ИТ-ландшафт проекта. Таким образом, создается основа для автоматического входа в соответствующие системы из SAP Solution Manager с целью настройки системы по концептуальному проектуу или с целью тестирования.

С помощью SAP Solution Manager можно обеспечить целостность данных в процессе пользовательской настройки неоднородной ИТ-среды. Платформа SAP Solution Manager обеспечивает администрирование процесса настройки, копирование пользовательских настроек, а также упрощение проверок непротиворечивости. SAP Solution Manager снижает объем ручных операций по синхронизации путем одновременного автоматического распространения пользовательских настроек на различных системах и централизованного управления всеми запросами на синхронизацию настроек.

Платформа SAP Solution Manager также представляет собой среду для анализа, дизайна и конфигурирования персональных бизнес-процессов. Решение SAP Solution Manager позволяет:

  • Определить все бизнес-процессы компании и использовать заранее определенные сценарии    внедрения решений SAP;

  • Хранить документацию в форматах PPT, XLS, DOC и т.д.;

  • Выполнить настройку бизнес-процессов, распределенных по нескольким системам SAP;

  • Провести тестирование сконфигурированных бизнес-процессов.

        Организация тестирования

       Платформа предоставляет единую точку доступа ко всей системной среде и обеспечивает централизованное хранение материалов и результатов тестирования. SAP Solution Manager содержит инструмент Test Organizer (Организатор тестирования). Он обеспечивает поддержку различных вариантов тестирования: от выполнения контрольных примеров до интеграционных сценариев тестирования, которые обрабатывают распределенные бизнес-процессы в рамках всего решения, а также предоставляет комплексную отчетность по тестированию.

        Инструмент Test Organizer включает в себя тестовые сценарии, выполняемые вручную или автоматически, такие как компьютерные средства тестирования (Computer Aided Test Tools – CATTs) и eCATTS. Наряду с тестовыми сценариями, предоставляемыми для определенных решений mySAP Business Suite, существует возможность определить собственные сценарии тестирования и включить в них существующие сценарии тестирования.

        Глобальное развертывание

        Холдинговые компании сталкиваются с необходимостью унифицировать свои основные бизнес-процессы, а также основные данные и организационные структуры, после чего координировать их в рамках всего ландшафта решения. С помощью SAP Solution Manager можно централизованно определить глобальные корпоративные бизнес-процессы (включая предварительную конфигурацию, документацию по процессам и предварительно определенные тестовые сценарии, относящиеся ко всей структуре холдинга), а затем сформировать на их базе корпоративный шаблон. В дальнейшем развертывание системы на региональных рынках можно начинать с внедрения корпоративного шаблона, а затем сконфигурировать систему исходя из локальных требований.

Функции управления версиями позволяют управлять выпуском версий шаблонов и обеспечивают основу для оптимизации глобальных процессов. В этом сценарии также можно использовать предварительно определенное бизнес-содержимое из центрального каталога бизнес-процессов SAP.

        Мониторинг и оптимизация решения

После запуска системы в эксплуатацию важно обеспечить поддержание работоспособности и производительности используемых решений. Платформа SAP Solution Manager обеспечивает полный интегрированный мониторинг бизнес-процессов и систем на основе полного документирования среды решений и базовых бизнес-процессов.

Мониторинг решений включает контроль за системой и бизнес-процессами, а также представление отчетов об уровне сервиса, что помогает отслеживать и администрировать все используемые решения. Предупреждения о нештатных ситуациях или возможности их возникновения в SAP Solution Manager отображаются в графическом виде с указанием точного места возникновения проблемы в ландшафте решения. С помощью щелчка мыши можно получить доступ к подробной информации о проблеме, а также о способах ее решения и услугах, которые помогут устранить возникшую проблемуу.

С помощью системы отчетов SAP Early Watch пользователь может самостоятельно формировать отчеты по статусу работы системы. Пользователь сам определяет содержимое создаваемых отчетов и временные интервалы их выполнения.

Пакет сервисов SAP Solution Management Optimization (Оптимизация управления решением SAP) обеспечивает оптимальную работу решения SAP на протяжении всего его жизненного цикла. Этот пакет оптимизирует работу приложений и систем, обеспечивая тем самым максимальную эффективность ключевых бизнеспроцессов.

  1.3 Implementation Guide

        Конфигурация корпоративных бизнес-процессов осуществляется в среде, называемой "Руководство по внедрению" (Implementation Guide IMG). Все бизнес-процессы компании могут внедряться в функциональность системы посредством конфигурации параметров в IMG.

        Implementation Guide (IMG) SAP играет самую важную роль в настройке системы SAP под требования конкретной компании. Настройка осуществляется посредством конфигурации программных продуктов SAP в то время, как базовая система SAP остается в неприкосновенности.

        Существуют следующие виды Руководства по внедрению (см. рис. 1.6)::

   

Рисунок 1.6 – Виды IMG.

  • Справочное IMG (Reference IMG);

  • IMG по предприятию (Enterprise IMG);

  • IMG по проекту/проектам (Project(s)IMG).

        Справочное (ссылочное) IMG – это стартовое Руководство по внедрению, которое содержит базовый набор опций конфигурации, с помощью которых можно получить любую производную функциональность SAP в зависимости от специфических требований. Это самая общая версия Руководства по внедрению, все остальные версии – производные от нее. Иерархия директорий Ссылочного IMG показана на рис. 1.7.

   

Рисунок 1.7 – Ссылочный IMG.

    IMG по предприятию – это сокращенный вариант Ссылочного IMG и представляет только ту функциональность, которая необходима конкретному предприятию. Относящиеся к исключенным модулям опции конфигурации отфильтровываются во время генерации IMG по предприятию..

     Так как внедрение проекта SAP проходит поэтапно, IMG по предприятию создается отдельно для каждого из таких этапов, полученные руководства называются Проектными IMG (IMG-проект). Все конфигурации для конкретного проекта осуществляются в IMG через соответствующее Проектное IMG, причем доступ к IMG по предприятию и Ссылочному IMG ограничивается с целью исключения возможности внесения в них случайных изменений. В случае необходимости, рамки Проектного IMG могут расширяться – именно поэтому такая архитектура оказывается чрезвычайно удобной для управления настройками. В оставшейся части этого раздела речь пойдет только о Проектном IMG.

Проектный IMG. Транзакции конфгурации

    Проектный IMG содержит множество транзакций конфигурации, которые являются средствами настройки различных процессов или функциональностей, предоставляемых SAP. Конкретный бизнес-процесс может настраиваться с помощью нескольких транзакций конфигурации. По сути, каждая транзакция конфигурации – это одна из 8000 таблиц параметров, предусмотренных в SAP. Это могут быть такие параметры, как коды компании, счета Главной книги, диапазоны номеров счетов, контрольные коды счетов дебиторской или кредиторской задолженности, коды записи, коды местоположений, категории поставщиков или потребителей, коды налогов, фискальные календари, валюты и факторы обменного курса, диапазоны номеров бухгалтерских транзакций и т. д. Для каждой транзакции конфигурации предусмотрен текст со справкой (помощью) для конкретной транзакции, ее назначение и влияние на функциональность того или иного процесса.

  Проектное IMG организовано по иерархическому принципу – различные транзакции конфигурации сгруппированы в иерархии директорий, которые отражают порядок внесений настроек в тот или иной модуль (см. рис. 1.8). На уровне модулей директории IMG упорядочены в соответствии с порядком внедрения модулей SAP. Впрочем, некоторые параметры или транзакции конфигурации не зависят от конкретных модулей, и затрагивают все (или почти все) модули и функциональности системы. Такие транзакции конфигурации, которые называются Общие установки, размещаются на верхнем уровне иерархии IMG. Таким образом, порядок размещения призван предотвратить появление громадного одноуровневого списка транзакций конфигурации, который был бы совершенно неуправляем.

   

 Рисунок 1.8 – Иерархия директорий Ссылочного IMG.

Статус конфигурации может обновляться на любом уровне иерархии Проектного IMG. В SAP предусмотрены стандартные значения статусов – такие, как «завершенный» и «незавершенный» (значения статусов также могут быть созданы пользователем), их необходимо обновлять вручную.

    Документация по конфигурации обеспечивает запись аннотаций, тем и проблем по различным установкам конфигурации для каждой транзакции конфигурации. Эта документация обычно создается в свободной форме, но ее можно стандартизировать на основе требований различных проектов..

    В дальнейшем Проектный IMG будем называть "IMG-проект".

Использование IMG для внесения настроек в SAP

    Для любого проекта внедрения SAP существует небольшой набор изначальных параметров или транзакций конфигурации, которые необходимо задать:

  1. Клиент (Client) – cамый высокий организационный уровень в SAP R/3

  2. Код компании (Company code) – самое мелкое юридическое образование, необходимое для внешней отчетности, годового баланса, выписок из Книги закупок и т. д. Клиенту может быть присвоено несколько кодов компании, но никогда наоборот.

  3. Параметры для настройки  бухгалтерского учета:

  • План счетов (Chart of accounts) – необходим для внешней отчетности. У каждого клиента может быть только один план счетов, причем все коды компании внутри клиента должны использовать именно этот план счетов.

  • Область контроля кредита (Credit control area) – организационная структура, которая используется для управления и контроля кредита, а также для отчетности.

  • Области бизнеса (Business areas) – применяются для гибкой финансовой отчетности, в случае необходимости создания отдельного годового баланса, или отчета о прибылях и убытках, обычно используется в США.

  • Область контроллинга (Controlling area)  – высшая  организационная cтруктура для внутренней отчетности и бухучета, которой может быть присвоен один или несколько кодов компании.

  • Единица учета результатов (Operating concern) – каждой области контроллинга присваивается только одна единица учета результатов, которая используется при наличии модуля анализа прибыльности. Не все клиенты SAP внедряют этот модуль.

  • Счет Главной книги (General Ledger account) – применяется для внешней (обязательной) отчетности с использованием плана счетов.

  1. Завод (Plant) – единица хранения, учета, обработки или производства. Коду компании или закупающей организации может быть присвоено несколько заводов.

  2. Параметры для настройки процессов закупок и продаж:

  • Организация продаж (Sales organization) – высший уровень управления продажами и соответствующей отчетности.

  • Канал продаж и дистрибуции (Sales distribution channel) – характеризует различные режимы поставок конечным клиентам (дистрибуторам, агентам, точкам розничной торговли, интернет-магазинам и т.д.). Организации продаж может быть присвоено несколько каналов продаж и дистрибуции.

  • Подразделение продаж (Sales division) – используется для управления группой продуктов и отчетности по ним. Несколько подразделений продаж может быть присвоено нескольким организациям продаж или каналам продаж и дистрибуции.

  • Область продаж (Sales area) – используется для гибкого управления и отчетности; это комбинация нескольких организаций продаж, каналов продаж, дистрибуции и подразделений продаж.

  • Организация закупок (Purchasing organization) — организационная единица для осуществления закупок и создания заказов на закупку.

    1. Места хранения (Storage locations) – физическое место, где хранятся все учетные единицы компании, нескольким заводам может быть присвоено несколько мест хранения.

    2. Банк и Банковский счет (House Bank and Bank Account) – организационные структуры, которые отражают банковские отношения компании и ее банковские счета.

    3. Основные данные по поставщикам (Vendor Master) – важная информация о поставщиках. Например, название, адрес, номера телефонов и факсов, контактное лицо, информация о закупках, банковская и бухгалтерская информация. Эти данные сохраняются в полях транзакций, причем при выполнении транзакции эти данные могут использоваться по умолчанию.

    4. Основные данные по потребителям (Customer Master) – схожи с основными данными по поставщикам, содержат информацию о клиентах компании – такую, как название, адрес, покупательские предпочтения, маркетинговая и бухгалтерская информация. Когда выполняется транзакция, эти данные могут использоваться по умолчанию.
