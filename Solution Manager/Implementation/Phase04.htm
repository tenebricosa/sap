Введение

        На данном этапе ответственность за выполнение бизнес-процессов и техническую инфраструктуру берет на себя служба поддержки. Вся соответствующая документация оформлена и передана проектной командой соответствующим группам поддержки.

        В SAP Solution Manager этот процесс поддерживается посредством транзакции SOLUTION_MANAGER, в которой выполняется автоматический перенос из фазы внедрения в фазу использования решения. Например, структура бизнес-процесса в фазе внедрения является входными данными для функции контроля бизнес-процессов в фазе использования решения.

        Для данной фазы мы рассмотрим процесс создания электронных учебных материалов.

Создание учебных материалов. Учебные карты       

        При внедрении новых решений и функций конечные пользователи должны получать знания о новых функциях, которые они будут использовать. С помощью системы SAP Solution Manager на основе существующей структуры проекта создаются учебные карты по задачам конечного пользователя, например, новым функциям, которые будет использовать закупщик в компании.

        Учебная карта представляет собой структурированное оглавление интерактивного курса для самостоятельного изучения, содержащее краткую информацию о рассматриваемых темах и ссылки на учебный материал в формате HTML. В курсы самообучения можно внести материалы электронного обучения, например, обучающие программы SAP.

        Структура учебной карты приведена на рисунке 2.21.

Рисунок 2.21  – Структура учебной карты

        Для создания учебной карты необходимо сначала определить роли для тех групп пользователей, которым требуется впоследствии отправить эти учебные карты, например, закупщика или бухгалтера. В SAP Solution Manager используются деловые роли Должность и Организационная единица из HR. Если эти деловые роли не определены, их необходимо создать.

        На фазе реализации, а именно в конфигурации выполняется присвоение учебного материала структурным элементам, например, материала для электронного обучения, такого как обучающие программы SAP, презентации или контрольные примеры. Здесь же можно присвоить созданные роли структурным элементам для получения возможности фильтрации структуры проекта по группам пользователей (ролям), для которых предназначены определенные учебные разделы.

        По окончании тестирования проекта создаются учебные разделы на основе структуры проекта и присвоенного ей учебного материала, также на его основе создаются учебные карты в формате HTML для различных групп пользователей (ролей) в транзакции. Ссылки на учебные карты рассылаются конечным пользователям.
