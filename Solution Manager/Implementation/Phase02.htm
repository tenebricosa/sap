Введение      

        Цель этой фазы состоит в создании концептуального проекта, который является подробной документацией по требованиям к бизнес-процессам компании. В концептуальном проекте описываются общие принципы отражения бизнес-процессов в одной или нескольких SAP-системах. В нем подробно описывается назначение бизнес-сценариев, бизнес-процессов, шагов процессов и требования по внедрению решения SAP.

        Концептуальный проект является необходимой предпосылкой для перехода к фазе реализации, во время которой выполняется конфигурация и тестирование бизнес-процессов. Все планы тестирования базируются на структуре концептуального проекта. В процессе создания плана тестирования транзакции, присвоенные шагам процесса в концептуальном проекте, помещаются в планы тестирования и выполняются в качестве  функциональных тестов, проводимых с целью проверки выполнения транзакций.

Концептуальный проект

        Концептуальный проект включает в себя следующие структурные элементы, организованные в виде иерархии:

  • Организационные единицы (Organizational Units);

  • Основные данные (Master Data);

  • Бизнес-сценарии (Business Scenario);

  • Бизнес-процессы (Business process);

  • Шаги процессов (Process Step).

    Данные в концептуальном проекте, например, проектная документация, пакеты настройки бизнес-конфигурации или транзакции, присваиваются отдельным структурным элементам в SAP Solution Manager.       

        На уровнях ниже уровня бизнес-сценариев можно создавать структурные элементы для организационных единиц и основных данных. Эти структурные элементы используются лишь в том случае, если организационные единицы и основные данные релевантны только для бизнес-процесса, расположенного в структуре на более высоком уровне (см. рис. 2.11). Все уровни этой структуры можно изменять, что позволяет создать решение, полностью адаптированное к требованиям заказчика.

           

Рисунок 2.11 – Структура стандартного сценария

        Стоит  напомнить, что при создании концептуального проекта заказчики могут создать свои процессы, а могут выбирать предварительно сконфигурированные бизнес-процессы из Репозитария бизнесс-процессов. В зависимости от выбранного вида проекта и объема проекта будет доступно различное содержимое репозитария.

         Документ концептуального проекта также можно распечатать.

Рисунок 2.12 – Определение концептуального проекта

        При определении концептуального проекта документируются бизнес-процессы, которые должны быть отражены в системе SAP. Создается структура проекта, в которой релевантные бизнес-сценарии, бизнес-процессы и шаги процессов организованы в иерархическую структуру (см. рис. 2.6). Также существует возможность создания проектной документации и присвоения созданных документов отдельным сценариям, процессам или шагам процессов.

        В SAP Solution Manager существуют следующие возможности создания или изменения структуры проекта:

  • Добавление к структуре проекта новых организационных единиц, основных данных, сценариев, бизнес-процессов и шагов процессов;

  • Удаление из проекта или системной среды структурных элементов, определенных при помощи операций SAP Solution Manager, или выбор этих элементов из репозитария бизнес-процессов и помещение в структуру проекта;

  • Корректировка имен структурных элементов в зависимости от клиента;

  • Удаление из структуры проекта созданного структурного элемента;

  • Перемещение структурных элементов. Перемещение структурных элементов может осуществляться только в пределах того же уровня. Например, перемещение шагов процессов выполняется только на уровне шагов процессов.

        Существующий структурный элемент можно скопировать и затем вставить из буфера обмена. Это упрощает одновременное использование двух или более элементов в одном процессе или бизнес-требовании, например, в планировании логистической цепочки, где технология производства продукта на склад может отличаться от процесса единичного производства по заказу клиента, а бизнес-требования к отгрузке и биллингу могут быть одинаковыми независимо от продукта. Функция копирования доступна для общей и проектной документации, но копирование присвоенных управляющих данных и проблем невозможно.       

        Для определения способов выполнения бизнес-процессов в системах SAP необходимо присвоить каждой задаче процесса соответствующие транзакции.

        Шагам процесса можно присвоить следующие объекты:

  • Транзакции;

  • Программы;

  • Предварительно определенные URL из списка;

  • Веб-адрес или файл.

        В настоящее время можно присвоить даже те транзакции и программы, которые еще не созданы. Это может оказаться полезным при необходимости ввода всей информации о данном этапе проекта в текущий момент, тогда как выполнение этих транзакций или программ может быть отложено на более позднее время.

        Для организационных единиц, основных данных и задач процессов необходимо указать компонент в системной среде, которому принадлежит структурный элемент. Для бизнес-процессов указать компонент невозможно, поскольку некоторые бизнес-процессы проходят через различные компоненты. Присвоение компонентов выполняется на уровне задач процессов.

        При выборе в Репозитарии бизнес-процессов предварительно определенного структурного элемента его компоненты определяются автоматически.

        В SAP Solution Manager содержатся графические представления бизнес-процессов. С помощью них можно узнать какие задачи содержатся в процессе, в каких логических компонентах выполняются задачи процесса,  какие зависимости существуют между задачами процесса (отношения "предшественник-преемник"). Можно не только просматривать процессы, но и изменять связи между элементами графики.

        Для всех присвоений, выполненных на этапе создание концептуального проекта, существует возможность создания журнала использования.

 Создание и печать документа концептуального проекта      

        После создания концептуального проекта (структуры проекта, документации, присвоений транзакций) можно распечатать документ концептуального проекта для утверждения или дальнейшего использования. Для шаблонного проекта можно выбрать общую или проектную документацию. Существует возможность интеграции общей документации в концептуальные проекты, например, для ее распространения при глобальном развертывании.

        Перед созданием документа концептуального проекта необходимо выбрать элементы, включаемые в документ  концептуального проекта, и элементы для вывода на печать. Например, можно выполнить фильтрацию по виду или по статусу документа или выбрать элементы из структуры проекта. Если один документ встречается в концептуальном проекте несколько раз, то можно либо несколько раз включить документ целиком, либо вставить ссылку на уже включенный документ.

     Документ концептуального проекта может создаваться в режиме оффлайн или онлайн.

   

Рисунок 2.13 – Создание документа концептуального проекта.

        Во время создания документа в режиме онлайн SAP блокируется (рекомендуется только для небольших документов концептуального проекта). При создании документа проекта в режиме оффлайн данные загружаются из SAP-системы на жесткий диск, и в документе концептуального проекта запускается макрос создания (для любых документов концептуального проекта).

        Документ концептуального проекта можно распечатать при использовании функции печати текстового редактора. При помощи функции сохранения текстового редактора можно сохранить документ в локальном каталоге.

 Создание проектной документации

        Разделим всю документацию на этапе концептуального проектирования на (см. рис 2.14):

  • Общую документацию;

  • Проектную документацию.

   

Рисунок 2.14 – Документация.

        Общая документация – вся документация, поставляемая SAP. Это все документы и диаграммы сценариев. Эти документы включают в себя, например, описания сценариев, руководства по инсталляции, документацию по транзакциям. Имеется возможность сортировки или выгрузки документов, а также копирование документов для дальнейшего использования их в проектной документации.

        Проектная документация – это документация по отдельным структурным элементам, относящаяся к данному проекту. Порядок присвоенных документов в структуре также отражается в документе концептуального проекта.

        Всем создаваемым документам присваивается определенный вид документа. В зависимости от вида документа, вызывается соответствующий шаблон документа. Можно использовать только те виды документов, которые имеют либо образцы документов, либо расширения файлов, поддерживаемые в SAP Solution Manager. Если шаблон или расширение файла виду документа не присвоены, запускается текстовый редактор.

        Поставляемые виды документации зависят от определения проекта и его присвоений, выполненных в транзакции управления проектом на этапе Определение стандартов проекта.

Администрирование

        В администрировании концептуального проекта есть возможность ввода актуальных данных проекта по каждому структурному элементу в структуре проекта.

        Можно ввести или изменить следующие данные проекта:

  • Статус;

  • Плановые данные (начало, завершение, рабочие дни);

  • Фактические данные (начало, завершение, рабочие дни, оставшиеся ресурсы). В случае ввода дат, не принадлежащих периоду проекта внедрения, появится сообщение об ошибке; 

  • Присвоенные сотрудники проекта (присвоенные в процессе подготовки проекта);

  • Имя/описание.

        По умолчанию изменения данных проекта (например, изменения и удаления членов проектной группы и ключевых слов) не наследуется автоматически структурными элементами более низкого уровня.   Для такого рода передачи данных существует функция Массовые изменения.

        С помощью друой функции Наследование данных подпозициями можно передать измененные данные всем зависимым структурным элементам.

        Введенные данные применимы только на текущей фазе проекта. После завершения выполнения всех операций на  фазе концептуального проекта для данного структурного элемента, структурному элементу в этой фазе присваивается статус Завершено. В фазе реализации этому структурному элементу снова присваивается статус Открыто.

        Все сохраненные изменения фиксируются в специальном журнале.
