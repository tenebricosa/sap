Пример плана проведения объем-теста

Цель

 

Цель этого документа – предоставить пример плана проведения объем-теста в продуктивной среде SAP.

Объем-тест предназначен для того, чтобы подтвердить, что продуктивная среда SAP сможет обслуживать весь объем ответственных бизнес-процессов системы R/3 компании клиента.

Методология

· Для обеспечения точности результатов выполнение объем-тестов проводится на реальном продуктивно работающем аппаратном обеспечении.

· Соответствующий объем транзакций создаются с помощью инструментария тестирования и оперативного ввода пользователями.

· Каждая итерация объем-теста называется циклом.

· Каждый цикл содержит отдельный бизнес-процесс или связанные бизнес-процессы, тестирование которых нужно выполнить.

· Процедура управления изменениями должна тщательно контролироваться.

· Действия всех групп, участвующих в проведении объем-тестов, должны быть скоординированы с выполнением общей работы.

Процедуры объем-теста

Внимание: прежде чем приступать к выполнению объем-теста, выполните резервное копирование, чтобы после завершения теста выполнить восстановление.

 

1. Создание эталонных статистических данных

  • Необходимо записывать статистические данные, чтобы сравнивать предыдущие циклы тестирования с последующими циклами.

 

2. Выполнение объем-тестов

  • Статистические данные должны аккуратно собираться и записываться, чтобы можно было провести точное их сравнение по каждому из циклов.

 

  • Все группы должны взаимодействовать друг с другом, чтобы обеспечить согласованность и полноту выполнения тестов. К основным участникам тестирования относятся: группа приложений, техническая группа, группа разработок и сетевая группа.

 

3. Анализ результатов

  • Результаты по каждой из областей мониторинга должны быть записаны и сравнены. Эти результаты также включают впечатления конечных пользователей, технические статистические данные, статистические данные в области приложений и т.д.

 

  • На этом этапе после завершения каждого из циклов тестирования проводится совещание для сравнения и анализа результатов.

 

4. Выполнение настроек среды SAP, если необходимо

  • Для эффективного тестирования необходимо, чтобы изменения строго контролировались и записывались.

 

  • Решения о выполнении изменений принимает группа, проводящая объем-тесты.

 

При необходимости повторите процедуру.

 

Стандарты мониторинга

Интервалы для сбора статистических данных – 15 минут.
  • SAP R/3: транзакции в контролируемые буферы, системный журнал, сеть.

Транзакции: ST02, SM21, SM51, SRZL.

  • Инструментарий уровня ОС для мониторинга аппаратного обеспечения, производительности R/3, сети.

Транзакции: ST06, ST03.
  • Инструментарий базы данных для мониторинга ее производительности.

Транзакции: ST04.
