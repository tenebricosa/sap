Рассмотрение проекта внедрения R/3

Определение объема проекта - один из наиболее важных шагов в структурировании процесса, который должен начаться в ближайшее время. Объем проекта должен представлять собой письменный документ, в котором должно быть отражено:

  • что предстоит сделать;
  • временная перспектива, в которой работа должна быть выполнена;
  • сколько будет стоить достижение требуемых результатов.

Объем проекта должен включать цели и задачи проекта, а также устанавливать рамки работы проектной группы. При определении объема проекта по внедрению R/3 необходимо рассмотреть три вопроса:

Цели и задачи проекта

Очень важно, чтобы цели и задачи были четкими и измеряемыми. Следствием будет не только список всех ожидаемых результатов системы и проекта, но и основа для оценки сроков выполнения и возникающих затрат.

Среда ИТ

В основе системы R/3 лежит многоуровневая архитектура клиент/сервер. Она является открытой системой, то есть, разработана для использования в операционных системах различных производителей. Это обеспечивает отделу информационных технологий гибкость в разработке пользовательских решений по управлению информацией.

Бизнес-процессы

Бизнес-процессы, которые будут поддерживаться системой SAP R/3, определяются на основе потребностей клиента. Для просмотра ссылочной модели по процессам или компонентам может использоваться бизнес-навигатор. Для оценки бизнес-процессов, включенных в проект, необходимо заполнить документ по объему внедрения .

Необходимо четко определить объем проекта:

  • установить ожидаемые результаты;
  • определить поставляемые в рамках проекта результаты информацию;
  • задать критические факторы успеха;
  • задать объем выполняемой работы;
  • установить критерии управления и соответствия целям проекта.

Объем проекта служит для очерчивания границ работы, которую предстоит выполнить. После того, как объем проекта определен, можно воспользоваться системными требованиями для создания плана проекта, в котором детально изложены сроки, трудозатраты и исполнители проекта.

Объем проекта включает:

  • число организационно-экономических единиц (размер, география, сложность);
  • уровень запланированной общности в разных организационно-экономических единицах;
  • состояние интерфейсов и необходимость переноса данных;
  • количество процессов;
  • уровень организационных изменений (структура, отношения);
  • отделения компании или число предприятий;
  • направление организационно-экономической деятельности или виды продукции;
  • временные сроки;
  • оценка общих затрат;
  • присвоение сотрудников проекту;
  • организационное участие;
  • оказываемый эффект на предприятие (должности, квалификации);
  • реализуемые преимущества;
  • технические компоненты;
  • основные результаты в рамках проекта.

Для нумерации и классификации требуемых бизнес-процессов может использоваться ряд методов. Хорошей стартовой точкой в этом вопросе может стать обзор ссылочной модели R/3. После этого необходимо заполнить документ объема внедрения для учета бизнес-процессов и организационно-экономических факторов, включаемых в объем проекта. Этот документ может использоваться как концептуальный проект для четкого установления границ данной фазы проекта. Кроме того, на протяжении всего жизненного цикла проекта может проводиться контроль качества путем сравнения хода выполнения проекта с динамикой объема.

При подготовке документа по объему внедрения необходимо четко указать, какие процессы и сферы лежат вне объема проекта. Необходимо документально зафиксировать дату окончания и ключевые этапы проекта. Следует проанализировать цели и задачи проекта и обеспечить их реальность и контролируемость. Необходимо внимательно проверить документ на непротиворечивость. Ведь он будет использоваться для контроля направления и управления изменениями проекта. Документ по объему внедрения должен отражать рамки проекта, основанные на бизнес-факторах, целях проекта, существующей среде ИТ, ограничениях в сроках и затратах, и воздействии реализации проекта на предприятие.
