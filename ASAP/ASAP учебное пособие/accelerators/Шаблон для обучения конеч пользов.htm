Раздел 1. Введение

Данный план обучения и создания документации для конечных пользователей охватывает работу, предлагаемую группой по обучению и документации для поддержки проекта внедрения SAP по обучению и созданию документации для конечных пользователей. Данный план предоставляет детальное описание подхода к достижению целей проекта по обучению и созданию документации для конечных пользователей. Он служит в качестве основного принципа выполнения своевременной разработки обучения в рамках бюджета и будет использован для измерения эффективности проекта.

Элементы плана обучения и создания документации для конечных пользователей описаны в следующих разделах:

·        раздел 2, Задачи и трудовыеотношения, обобщает цели и задачи по разработке учебных материалов и идентифицирует ключевой персонал проекта;

·        раздел 3, Объем работы, предоставляет детальное описание выходных документов проекта;

·        раздел 4, Подход к разработке, описывает подход и операции, требуемые для составления каждого выходного документа проекта;

·        раздел 5, Календарный план проекта, предоставляет календарный план разработки конечных результатов проекта;

·        раздел 6, Предпосылки по проекту, обобщает предпосылки для выполнения работы, основанной на предлагаемом бюджете и календарном плане.


 

Раздел 2. Задачи и трудовыеотношения

2.1        Цель проекта

(ПРИМЕР ЦЕЛИ) Основная цель группы по обучению и документации представляет собой своевременную разработку эффективного обучения и документации для конечных пользователей для внедрения программного обеспечения SAP.

2.2        Задачи проекта

(ПРИМЕРЫ ЗАДАЧ) Группа по обучению и документации определила следующие ключевые факторы успеха для достижения задач обучения. Эти факторы успеха включают подготовку подхода к обучению и документации, который обладает следующими атрибутами:

·     высоким качеством,

·     является всесторонним,

·     основан на эффективности,

·     ориентирован на процесс,

·     сфокусирован на пользователе.

2.3        Трудовыеотношения

Руководитель проекта по обучению и документации является основным контактным лицом по обучению. Этот сотрудник отвечает за координацию мероприятий по разработке между разными членами группы внедрения SAP и за достижение задач обучения. Руководитель проекта по обучению и документации или предварительно назначенный представитель отвечает за получение утверждения всех выходных документов проекта, включая план обучения и создания документации для конечных пользователей, создание прототипов выходных документов проекта, разработку содержимого и организацию обучения.

Руководители по обучению и документации будут работать в тесном контакте с руководителями по бизнес-процессам группы внедрения для обеспечения технической точности всех учебных материалов.

В некоторых компаниях эти роли могут выполняться одним сотрудником.

Раздел 4. Подход к разработке

Этот раздел содержит подробную информацию о подходе к разработке и структурному плану проекта (СПП), который группа использует для поддержки решения по обучению и созданию документации для конечных пользователей. Этот документ не отражает порядок, в котором выполняются операции, а описывает каждую операцию и к какой фазе разработки она принадлежит.  Необходимо проанализировать документы операций в каждой фазе для просмотра задач, которые могут быть выполнены параллельно против зависимости от другой задачи.

4.1        Структурный план проекта разработки обучения

СПП на рисунке 4-1 иллюстрирует подход к разработке, которому следует группа по обучению и документации для проведения проекта. Каждая операция описана кратко ниже по тексту, как это иллюстрировано на рисунке 4-1 Структурный план проекта.  Более подробное объяснение по каждой операции и задачи, связанные с ней, представлены после рисунка 4.1.

·     Операция 1, Уточнение стратегии обучения и создания документации для конечных пользователей – определите стратегию обучения и создания документации, соотнесите членов группы внедрения и представителей конечных пользователей с целями проекта обучения и разработки документации для конечных пользователей и начните сбор информации для определения объема проекта разработки.

·     Операция 2, Анализ стратегии обучения и создания документации для конечных пользователей – разработка и завершение плана обучения и создания документации для конечных пользователей на основе стратегии обучения и задач.  Определите требования к документации и системе и учебный план.

·     Операция 3, Создание прототипов выходных документов обучения и документации для конечных пользователей – проектирование и подготовка примеров материалов, которые служат визуальной моделью содержания обучения и материалов документации. Разработайте учебный сценарий диалогового прототипа и включите план обновления базы данных обучения. На базе учебного сценария разработайте прототип (функциональную модель) диалоговой системы и включите образцы материалов. Представьте прототип и получите утверждение проекта. После анализа, достижения консенсуса и утверждения прототипа необходимо пересмотреть план обучения и создания документации для конечных пользователей и завершить оценку ресурсов и календарного плана. На основе прототипов необходимо проанализировать диалоговые требования к системе.

·     Операция 4, Определение содержимого обучения и документации для конечных пользователей, - задачи данной операции выполняются совместно с задачами Операции 5, Разработка содержимого обучения и документации для конечных пользователей.  На основе результатов создания прототипов необходимо разработать модели документации и стандарты для поддержки содержимого обучения и материалов документации.  Проведите детальные интервью для сбора информации по операциям.  Разработайте схемы курсов в подготовке разработки содержимого.  Установите процесс анализа, инициируемый в ходе разработки (операция 5).

·      Операция 5, Разработка содержимого обучения и документации для конечных пользователей, - шаги данной операции выполняются совместно с шагами операции 4, Определение содержимого обучения и документации для конечных пользователей.  Выполнение процесса анализа содержимого, как описано в процессе анализа, определенном в операции 4, Разработка моделей, стандартов и процесса анализа.  После проверки содержимого заполните базу данных среды обучения SAP и выполните тест используемости учебных материалов.

·      Операция 6, Завершение разработки содержимого обучения и документации для конечных пользователей и подготовка к проведению обучения – завершите разработку учебного плана и распределение функциональных ролей. Разработайте план материально-технического обеспечения обучения и проведите тестирование системы обучения.  Выполните обучение референтов и внедрите любые соответствующие улучшения для учебных материалов.  Разработайте содержимое для обучения перед продуктивной эксплуатацией.

·      Операция 7, Проведение обучения конечных пользователей перед началом продуктивной эксплуатации – закончите разработку плана материально-технического обеспечения обучения и проведите обучение конечных пользователей перед началом продуктивной эксплуатации.  Оцените мероприятия по обучению и включите изменения.

·      Операция 8, Проведение обучения конечных пользователей после начала продуктивной эксплуатации – оценка переноса знаний. Предоставьте обучение для конечных пользователей после начала продуктивной эксплуатации. Оцените эффективность обучения.

4.2        Подход к разработке

Проект обучения и создания документации для конечных пользователей состоит из восьми операций. Каждая операция и связанные с ней шаги описаны ниже.

Операция 1, Уточнение стратегии обучения и создания документации для конечных пользователей

Анализ стратегии внедрения проекта

Перед разработкой плана обучения и создания документации для конечных пользователей вырабатывается стратегия для обеспечения соответствия работ по разработке обучения целям внедрения SAP. В этот момент необходимо понять общую стратегию внедрения, ожидания ключевых участников и установить цели всего объема проекта.  Необходимо рассмотреть наличие ресурсов обучения и календарный план внедрения.  Дополнительно к идентификации ключевых ролей требуется внедрение успешной стратегии и соответствующих сфер ответственности. План и проект обучения и создания документации для конечных пользователей являются компонентами общего проекта внедрения SAP.

 

Оценка потребностей конечных пользователей в обучении и создании документации

Результаты оценки потребностей конечных пользователей в обучении и документации будут переданы целевой аудитории. Группа по обучению и документации должна рассмотреть открытые вопросы и обеспечить адекватность и корректность реакции.

Разработка плана проекта обучения и создания документации для конечных пользователей

План проекта обучения и создания документации для конечных пользователей идентифицирует задачу, этапы и ресурсы для всего проекта обучения и создания документации для конечных пользователей.  Этот план обеспечивает выполнение сроков, обеспечение качества и соблюдение бюджета проекта.

Разработка рекомендаций по обучению и созданию документации для конечных пользователей

В ходе этой задачи начинается исходная разработка плана по обучению и документации для конечных пользователей.  Исходный объем, цели и задачи проекта обучения и создания документации для конечных пользователей документируются, включая возможности решений по обучению и созданию документации для конечных пользователей, затраты и ресурсы. Этот план служит в качестве управляющего и ссылочного документа для проектной группы и является основой для отслеживания и отчетности по динамике реализации проекта. Кроме того, план содержит задачи проекта, требования к ресурсам, подход к разработке, календарный план и предпосылки проекта.

 

Проведение совещания по уточнению проекта обучения и создания документации для конечных пользователей

Совещание по уточнению проекта должно быть проведено в начале проекта. Группа по обучению и документации встречается с представителями группы внедрения и конечных пользователей для анализа и утверждения целей и задач обучения, создания документации и обеспечения понимания стратегии и инструментов. Группа по обучению и документации также обсуждает предварительный объем разработки и ресурсов, включая прогнозируемое время, которое потребуется группе внедрения для анализа и проверки выходных документов. Календарный план включается в план обучения и создания документации для конечных пользователей, а в ходе совещания по уточнению утверждается целевая группа для оценки потребности конечных пользователей в обучении и разработке документации.  

 

Операция 2, Анализ стратегии обучения и создания документации для конечных пользователей

Обновление данных оценки потребности конечных пользователей в обучении и создании документации для

На основе актуальной информации необходимо обновить данные потребности конечных пользователей в обучении и документации с учетом дополнительной информации по объему.

Определение требований к обучению и созданию документации для конечных пользователей

На основе рекомендаций, определенных в задаче Разработки рекомендаций к обучению и созданию документации для конечных пользователей, Оценки потребности конечных пользователей в обучении и документации, а также на основе информации совещания по уточнению проекта будет разработано решение, включающее аспекты обучения и создания документации для каждого необходимого вида содержимого обучения и материалов документации.

Требования к обучению будут определены совместно с решением по документации.  Требования по обучению включают метод проведения обучения (аудиторные материалы на бумажной основе, компьютеризированное обучение CBT), число конечных пользователей, число и длительность курсов и количество упражнений.  Форматы для учебных материалов, включая подробный уровень, указаны в задаче Определения требований к документации конечных пользователей.

Проектирование учебного плана и тем документации

Проектная группа разработает учебный план, который будет служить в качестве концептуального проекта для разработки обучения. Проект учебного плана поможет обеспечить разработку только требуемых материалов.  Учебный план разрабатывается на основе оценки потребности конечных пользователей в обучении и документации.  Отдельный документ учебного плана отражает исходные названия должностей конечных пользователей и обеспечивает матрицу требуемой функциональности SAP и необходимого обучения в зависимости от должности.  Кроме того, создается список тем документации для идентификации важных системных процедур или транзакций, которые должны включаться в план разработки.

Определение требований к среде обучения SAP

Проектная группа определяет требования к среде обучения SAP для поддержки проекта по обучению и разработки документации.  Выполняется оценка числа инстанций и мандантов в дополнение к созданию плана обновления для среды обучения.  Эта информация служит в качестве основы для установки этих систем в соответствующее время реализации проекта, а также для указания разработчикам обучения и документации, в какой среде и в какое время они должны работать.

Создание плана обучения и ведения документации и управление знаниями

Проектная группа разрабатывает план для идентификации потребностей в знаниях группы разработки и сотрудников ведения документации и организует их обучение методам ведения информации. Определите, как будет обеспечиваться поддержка конечных пользователей на краткосрочный и долгосрочный период в течение жизненного цикла SAP.

Завершение разработки плана обучения и создания документации для конечных пользователей

Проектная группа учитывает все согласованные замечания и заканчивает план обучения и создания документации для конечных пользователей. Окончательный план будет представлен на утверждение всем ключевым участникам проекта. Критически важным является утверждение плана всеми ключевыми участниками, включая форматы в прототипе перед началом разработки обучения.

Операция 3, Создание прототипов выходных документов обучения и документации для конечных пользователей

Проектирование прототипов обучения и документации для конечных пользователей

Проектная группа составляет общую структуру содержимого обучения и документации и функционального проекта высокого уровня.  Операции проектирования разрабатываются на основе утвержденного документа требований для содержимого обучения и документации.  При необходимости должны быть выбраны и разработаны инструменты создания содержимого обучения и пользовательской автоматизации/преобразования.  Конкретные операции включают следующее:

·     Изучение технической осуществимости предложенных решений с учетом аспектов инфраструктуры.

·     Создание функциональной модели предлагаемого содержания и материалов документации.  Прототип содержимого обучения и документации позволяет оценить требования к интерфейсу конечного пользователя, функциональности и технической осуществимости проекта.  Это важный шаг в обеспечении согласованности представлений конечных пользователей, группы обучения и разработки документации и группы внедрения по виду и функциям окончательного варианта содержимого обучения и документации.

·     Планирование и проектирование процессов присвоения полномочий, стандартов и подпрограмм переноса.

Анализ системных требований

Группа по обучению и документации начнет процесс разработки диалоговой документации с создания выходных документов прототипа. Будут определены требования к системе и разработан учебный сценарий для демонстрации предполагаемого подхода и получения поддержки со стороны конечных пользователей.  Конкретные операции включают:

·     оценку организационной структуры компании и конечных пользователей для определения функциональности системы;

·     изучение содержания проекта обучения и документации для конечных пользователей для определения требований к содержанию обучения и документации;

·     помощь проектной группе информационных технологий в анализе технической инфраструктуры  и определении требований к системе;

·     Создание учебного сценария для содействия в обсуждении требований к системе и функциональности.  Учебный сценарий является серией слайдов или диаграмм, которые отражают, какие виды материалов будут предоставляться, как организован материал, и как пользователи смогут выполнять навигацию к материалам, которые им необходимы.  Этот выходной документ обеспечивает поддержку в обсуждении и уточнении между группами внедрения.

·     Подготовка и получение утверждения документа требований при внедрении диалоговой системы.  Документ требований будет содержать информацию по тому, как решение интегрировано с существующими техническими архитектурами на всех объектах конечных пользователей, а также характерные особенности методов навигации и доступа к разным видам материала и сведения по другим функциональностям системы.

Представление прототипов обучения и документации для конечных пользователей

Группа по обучению и созданию документации представляет прототип для анализа. Этот анализ должен быть сосредоточен на формате, последовательности учебных материалов и глубине развертки – т.е. нефактическом содержимом, – представленных в прототипе. По причине того, что все будущие учебные материалы будут основаны на прототипе, методы и формат информации должны быть согласованы со всеми участвующими сотрудниками.

Операция 4, Определение содержимого обучения и документации для конечных пользователей

Разработка шаблонов, стандартов и процесса анализа

Группа по обучению и документации разработает шаблоны документации и стандарты, которые будут служить в качестве справочного руководства для всей группы по обучению и документации для обеспечения согласованности материалов. Руководитель по обучению и документации создает, ведет и обеспечивает выполнение условий, установленных стандартами документации.  Разрабатывается процесс анализа и расписание циклов анализа, включая роли и сферы ответственности.

Определение и внедрение процесса анализа и усовершенствования

Группа по обучению и документации создает и внедряет процесс анализа эффективности проекта обучения и создания документации для конечных пользователей.  Он будет включать то, как внедрить функцию обратной связи и усовершенствовать обучение для будущих слушателей и мероприятий.  Необходимо передать информацию о процессе участникам проекта и соответствующему персоналу.

Разработка процесса сбора данных

Группа по обучению и документации собирает дополнительные данные для каждой транзакции и бизнес-процесса, используя данные, которые ранее были собраны в ходе фазы уточнения. Группа по обучению и документации проводит интервью с конечными пользователями и специалистами по предмету, а также с членами группы внедрения для сбора любых отсутствующих данных. Это помогает обеспечить понимание внедрения, бизнес-процессов и задач должностей перед разработкой учебных материалов.

Структурированные интервью по проекту проводятся с определенными специалистами по предмету. Группа по обучению и документации проводит интервью со специалистами по предмету и представителями конечных пользователей следующих областей (как минимум):

·        приведите список процессов и подпроцессов.

 

 

Интервью будут сосредоточены на сборе следующих видов информации:

·     Какова концепция успеха и в чем возможности для усовершенствования?

·     Что является будущим процессом и как он сравним с актуальным процессом?

·     Какие знания и навыки необходимы для достижения максимальной производительности?

·     Какими знаниями и навыками обладают в настоящий момент пользователи?

Группа по обучению и документации совещается со специалистами по предмету и проводит анализ существующей документации для сбора информации о системе и связанных с системой операциях (таких как операции вручную или прежняя система). Цели состоят в определении бизнес-процессов и рабочих процессов, указании основных результатов, за которые отвечает сотрудник соответствующей должностной функции и подтверждении или идентификации должностной деятельности, необходимой для достижения поставленных целей. Ключевые атрибуты, которые должны быть определены во время этого анализа, следующие:

·     предпосылки для операций,

·     частотность,

·     приоритет,

·     сложность,

·     последовательность ошибки.

 

Разработка и проведение семинара для разработчиков курсов

Группа по обучению и документации разрабатывает и проводит семинар для передачи информации о шаблонах, стандартах и формате содержимого разработчикам обучения и документации.

 

Разработка структуры курсов

Группа по обучению и документации разрабатывает подробную структуру курсов таким образом, чтобы каждый раздел курсов обучения включал:

·     аудиторию обучения,

·     задачи производительности,

·     предпосылки,

·     продолжительность курса,

·     ключевые моменты, которые необходимо подчеркнуть в концепциях,

·     требуемую последовательность обучения,

·     описание практических операций.

 

 

Операция 5, Разработка содержимого обучения и документации для конечных пользователей

Разработка содержимого документации

Группа по обучению и документации разрабатывает материалы документации для каждого курса. Формат курсов будет основан на утвержденном прототипе. Материалы документации могут соответственно состоять из следующего:

·     системы диалоговой справки,

·     документации по бизнес-процессам,

·     документации по транзакциям (ПБП),

·     кратких справочных руководств,

·     информационных заметок.

 

В разработке и анализе каждого вида материала группа по обучению и документации выполняет следующее:

·     анализ любых существующих концептуальных материалов и идентификацию материалов для повторного использования;

·     анализ существующей документации, выполнение диалоговых транзакций и работу со специалистами по предмету для идентификации содержимого и уровня детализации;

·     разработку материалов, которые предоставляют информацию о бизнес-процессах, описание должности, терминологию и описание должностных функций соответственно;

·     внутренний технический и редакторский анализ каждого документа;

·     предоставление готовых материалов предварительно назначенным специалистам по предмету для технического анализа на предмет соответствия и наличия необходимой бизнес-информации.

Разработка содержимого обучения

Группа по обучению и документации разрабатывает содержимое обучения для каждого курса. Формат курсов будет основан на утвержденном содержимом обучения. Материалы документации могут состоять соответственно из следующего:

·     концепций,

·     CBT – родовой обзор,

·     моделирования,

·     процедур конечного пользователя и процедур справочного руководства,

·     упражнений,

·     справочников референта,

·     курсов обучения референтов (обучение навыкам проведения курсов),

·     инструментов анализа.

 

В разработке и анализе каждого вида материала группа по обучению и документации выполняет следующее:

·     анализ любых существующих концептуальных материалов и идентификацию материалов для повторного использования;

·     анализ существующей документации, выполнение диалоговых транзакций и работу со специалистами по предмету для идентификации содержимого и уровня детализации;

·     разработку материалов, которые предоставляют информацию о бизнес-процессах, описание должности, терминологию и описание должностных функций соответственно;

·     внутренний технический и редакторский анализ каждого документа;

·     предоставление готовых материалов предварительно назначенным специалистам по предмету для технического анализа на предмет соответствия и наличия необходимой бизнес-информации.

Анализ содержимого

Группа по обучению и документации первоначально анализирует работу и предоставляет все содержимое обучения и материалы документации для анализа специалистам по предмету.  Материалы должны обновляться по мере получения результатов редакторской правки.  Группа по обучению и документации должна получить окончательное утверждение для всего содержимого обучения и материалов документации.

Загрузка данными среды обучения SAP

Группа по обучению и документации и группы внедрения определяет количество данных обучения, которые будут созданы для каждого курса на основе максимального количества слушателей курса, расписания экспериментального курса и частоты обновления и перенастройки манданта обучения. Заполните основного манданта обучения необходимыми данными для всех упражнений экспериментальных курсов. Часть этой операции должна состоять в документировании данных в формате, используемом референтом во время чтения курса.

Тестирование используемости содержимого

Группа по обучению и документации тестирует используемость содержимого обучения и материалов документации.  Аудитория будет включать представителей конечных пользователей, специалистов по предмету и ключевых пользователей. 

Операция 6, Завершение разработки обучения и документации для конечных пользователей и подготовка к проведению обучения

Завершение создания учебного плана и определения функциональных ролей

Группа по обучению и документации завершает этот процесс уточнением учебного плана относительно функциональных ролей.  Совместно с ответственными за бизнес-процессы, ИТ, группой по безопасности и полномочиям, группой управления изменениями и группой конфигурации необходимо идентифицировать любые известные ''окна" в учебном плане или дополнительные роли, которые не были идентифицированы.  Необходимо получить окончательное утверждение и передать информацию соответствующему руководству и членам проектной группы.

Разработка плана материально-технического обеспечения обучения

На основе плана обучения необходимо разработать план по референтам, аудиториям, календарному планированию, доступу к SAP и диалоговым системам, а также по оборудованию. Разработайте план тестирования для этих компонентов.

Тестирование системы обучения

Группа по обучению и документации продолжит процесс разработки содержимого обучения и документации, начиная с операции 3, Проектирование прототипов обучения и документации, и переходя непосредственно к разработке полной версии учебных материалов.  Конкретные операции включают:

·     тестирование для обеспечения согласованности всех технических компонентов и инструкций системы обучения;

·     расширение прототипов обучения и документации проекта до уровня среды, которая включает всю функциональность SAP, идентифицированную прототипом;

·     при использовании диалоговой документации необходимо подготовить процесс переноса для поддержки всего набора видов документов, которые будут при необходимости существовать в диалоговой системе поддержки;

·     преобразование каждого файла, который получил окончательный статус, и выполнение проверки качества каждого перенесенного документа до включения в общий Web-сайт.

Подготовка и проведение обучения референтов

Группа по обучению и документации подготавливает и проводит курсы обучения референтов для того, чтобы новые референты в ходе учебных мероприятий или семинаров получили навыки проведения обучения. Целью этих мероприятий является совершенствование способности референта проводить курсы обучения для конечных пользователей. После завершения обучения референты необязательно станут экспертами в соответствующих областях, тем не менее они будут обладать необходимыми знаниями и навыками, которые требуются для эффективного проведения курсов SAP.

Альтернативный план для выполнения этой задачи будет состоять в проведении обучения референтов непосредственно перед проведением экспериментального обучения.  Это будет способствовать включению референтов в процесс разработки на более раннем этапе и позволит им участвовать в экспериментальном проекте, получить дополнительные навыки и улучшить передачу знаний.  Проведение обучения референтов подобным образом позволяет задействовать их на более раннем этапе перед началом продуктивной эксплуатации, что отвлекает их от выполнения обычной работы.

Завершение разработки содержимого

После проведения тестирования используемости и обучения референтов будут собраны результаты обратной связи и закончена разработка учебных материалов. Отредактированные материалы будут состоять из диалоговых материалов и, по необходимости, материалов на бумажной основе для каждого курса. Создание базы данных для поддержки обучения будет также завершено.  

Операция 7, Проведение обучения конечных пользователей перед началом продуктивной эксплуатации

Внедрение плана материально-технического обеспечения обучения

Координатор обучения контролирует реализацию на данном этапе операции 6, Разработка плана материально-технического обеспечения обучения. Необходимо обеспечить наличие всех средств проведения и оценки аудиторного обучения, необходимых для организации обучения конечных пользователей.

Проведение обучения конечных пользователей

Референты проводят обучение конечных пользователей до продуктивного запуска системы.

Оценка эффективности обучения и внедрение усовершенствований

Группа по обучению и документации внедряет инструменты оценки, определенные в операции 4, Определение и внедрение процесса оценки и усовершенствования для анализа влияния обучения.

Операция 8, Проведение обучения конечных пользователей после начала продуктивной эксплуатации

Руководитель проекта по обучению и документации оценивает эффективность обучения конечных пользователей и идентифицирует области для усовершенствования. 

Проведение обучения конечных пользователей

Референты проводят обучение на основе результатов, полученных в операции 7, Оценка эффективности обучения и внедрение усовершенствований.  Дополнительное обучение должно быть сконцентрировано на областях, в которых конечные пользователи некорректно выполняют задачи или в которых требуется более глубокое обучение, или в областях, незатронутых в ходе обучения перед началом продуктивной эксплуатации.

Обеспечение поддержки непрерывного обучения и ведения документации и управление знаниями

Группа по обучению и документации должна закончить и передать для решения любые открытые вопросы лицам, ответственным за непрерывное ведение содержимого обучения и материалов документации.

Раздел 6. Рекомендации

Одним из ключевых элементов внедрения решения по обучению и созданию документации для конечных пользователей является предоставление подробного описания задач проекта, предпосылок и выходных документов. В разделе 3 данного документа описан объем работы, и в этом разделе представлены задачи и некоторые рекомендации, которые может быть необходимо внедрить для выполнения работы.

Трудовые отношения

·        Группа по обучению и документации будет работать в тесном контакте с группой внедрения для разработки решения по обучению и созданию документации для конечных пользователей, которое удовлетворяет задачам проектного обучения. Успех проекта в значительной мере зависит от своевременного предоставления исходных учебных материалов, работы назначенного персонала, быстрого анализа и утверждения проектируемых материалов со стороны специалистов по предмету (СП).

·        Основным контактным лицом для группы по обучению и документации является руководитель проекта по обучению и созданию документации для конечных пользователей.

·        Руководитель проекта по внедрению, директор проекта, спонсоры проекта и прочие участники принимают план обучения и создания документации конечных пользователей и выходных документов прототипа путем их утверждения. 

Наличие специалистов по предмету (СП)

·        В ходе проекта группа по обучению и документации должна по мере необходимости иметь доступ к сотрудникам, ответственным за соответствующие разделы выходных документов проекта. Руководитель проекта по обучению и документации определяет ожидаемую занятость каждого специалиста по каждой операции, как только будут определены требования. Руководитель обучения и документации для конечных пользователей определяет специалистов по предмету для каждого курса обучения.

·        Специалисты по предмету будут работать над проектом во время периодов сбора данных, определенных группой обучения и разработки документации и согласованных с руководителем проекта по внедрению. Вопросы сотрудничества со специалистами по предмету будут решаться руководителем проекта по внедрению.

·        Специалисты по предмету должны иметь достаточные знания и опыт, а также необходимые полномочия для принятия решений по содержимому учебных материалов в своих сферах ответственности. Календарный план проекта и бюджет могут повергнуться отрицательному воздействию в том случае, если специалисты по предмету не могут участвовать в запланированных интервью и анализе материалов.

·        Руководитель проекта по обучению и документации по необходимости будет планировать и координировать совещания и интервью со специалистами по предмету для поддержки работы группы по обучению и документации по разработке и обеспечит информирование участников о запланированных совещаниях и интервью.

Требования к доступу

·        В начале проекта будут идентифицированы любые аппаратные средства или программное обеспечение, требуемые для доступа к SAP, любые информационные системы или сети.

·        Перед тем как начать работу, группе по обучению и документации необходимо иметь доступ к системе SAP и сети LAN.

 

Информация о проектировании системы

·        Группе по обучению и документации необходим доступ к манданту SAP, отражающему актуальную конфигурацию системы. Календарный план, предлагаемый в данном плане обучения и создания документации для конечных пользователей, предполагает выполнение соответствующей работы по проектированию системы до того, как группа обучения и разработки документации начнет работу с учебными материалами. Если работа по проектированию не может быть завершена до запланированного начала работы группы по обучению и документации, то руководитель проекта по внедрению и руководитель проекта по обучению и документации должны согласовать вопрос о соответствующих средствах достижения целей обучения в рамках ограничений проекта. Как группа обучения и документации, так и группа внедрения должны осознавать, что изменения системного проекта в ходе разработки учебных материалов могут оказать влияние на объем проекта.

·        Для разработки требуется сконфигурированный и стабильно работающий мандант, загруженный необходимыми данными. Соответствующий персонал, ответственный за настройку и ведение данного манданта, должен быть определен в начале проекта. Мандант должен содержать основные данные и таблицы. Кроме того, необходимо настроить наборы функциональных данных. Эти наборы данных включают номера поставщиков, номера материалов, группы закупок, закупочные организации, группы счетов, места возникновения затрат, заявки, заказы на поставку и поступления материала, которые поддерживают мероприятия по разработке. Группе по обучению и документации необходимо обеспечить безопасность в работе по документированию всех транзакций для учебных материалов. Отсутствие соответствующих данных замедляет разработку и может повлиять на бюджет и календарный план разработки обучения. Специалисты по предмету отвечают за предоставление примеров реальных данных для включения в материалы обучения конечных пользователей.

·        Группа внедрения должна предоставить обновленный список процедур бизнес-процессов SAP (ПБП), бизнес-процессов, прежних и новых систем, как только они становятся доступными.

·        ПБП и прочая документация должны быть достаточны для выполнения указанных операций. ПБП должны отражать методологию компании для выполнения операций.

·        Группа по обучению и документации по возможности будет использовать информацию существующих материалов. Для содействия данному процессу группа внедрения предоставляет группе по обучению и документации доступ к всей существующей исходной и ссылочной документации — включая ПБП, документацию тестирования и презентации проекта. Группа по обучению и документации предполагает, что содержимое ссылочной документации является соответствующим и не изменится существенным образом в ходе проекта.

·        Для обеспечения технической точности материалов группа по обучению и документации должна информироваться о любых изменениях системного проекта и “будущих” рабочих процессах. Соответственно ключевые члены группы по обучению и документации должны присутствовать на совещаниях группы внедрения для того, чтобы быть в курсе системного статуса, изменений и проблем.  Это поможет избежать изменений объема проекта, задержек и изменений календарного плана.

Выходные документы проекта

·        Все выходные документы будут иметь формат, который соответствует требованиям к документации, и способствует их ведению определенным персоналом. Группа по обучению и документации, представители группы внедрения, спонсоры проекта и прочие участники должны согласовать формат для учебных материалов в начале проекта.  Это будет достигнуто благодаря использованию материалов прототипа, которые будут служить в качестве моделей для будущих усилий по разработке. Прототип должен быть проанализирован и утвержден представителями группы внедрения, спонсорами проектов и другими участниками. После утверждения прототипа последующие изменения проекта могут оказать влияние на календарный план и бюджет.

·         

·        Все учебные материалы будут разработаны при помощи соответствующего программного обеспечения и аппаратных средств. Это будет определено в ходе совещания по уточнению проекта и задокументировано в плане обучения и создания документации для конечных пользователей.

·        Учебные материалы будут разработаны на основе языковых требований и определены во время разработки плана обучения и создания документации для конечных пользователей.

·        Все материалы предоставляются при помощи средств обучения, определенных во время разработки требований к плану обучения и создания документации для конечных пользователей. Группа по обучению и созданию документации отвечает за копирование и распределение всех материалов для проведения обучения.

·        Процесс приемки окончательных выходных документов обучения определяется в начале проекта. Этот процесс инициируется руководителем проекта по обучению и документации по завершении разработки обучения.

Анализ выходных документов

·        Успех работы по разработке обучения в значительной мере зависит от возможности контактов со специалистами по предмету, быстрого технического анализа и утверждения проектных материалов.

·        Группа по обучению и документации использует процесс анализа и приемки, состоящий из текущего анализа и циклов изменений для всех учебных материалов.

·        В начале проекта должен быть определен процесс составления комментариев к анализу и обработки конфликтных комментариев аналитиков. Целью процесса анализа должна быть выработка окончательной версии документа, которая снова передается группе по обучению и документации.

·        Технический анализ должен быть завершен согласно календарному плану разработки обучения. Временные границы анализа должны быть согласованы в начале работы над проектом.  Непредвиденные задержки могут иметь место в том случае, если технический или редакторский анализ выходит за рамки согласованных временных границ.<. Руководитель проекта по обучению и документации или руководители групп по обучению и документации будут контролировать календарный план и заранее уведомлять руководителя проекта внедрения и руководителей групп внедрения о необходимости неожиданных или сокращенных циклов анализа.

Календарный план проекта

·        Между конфигурацией системы и разработкой обучения и документации существуют прямые отношения.  Любые изменения календарного плана группой внедрения приводят к соответствующему запаздыванию по срокам, к которым должны быть представлены необходимые выходные документы группы по обучению и документации. Поэтому важно, чтобы группа по обучению и документации являлась неотъемлемой частью группы внедрения. Группа по обучению и документации должна уведомлять группу внедрения о потенциальных изменениях календарного плана, когда они становятся очевидными.

·        Специалисты по предмету будут следовать утвержденному календарному плану для анализа документов и предоставления запрошенной информации.

·        Руководитель проекта по обучению и документации предоставит новую информацию по динамике разработки обучения, как это определено в начале проекта. Динамика будет отслеживаться по календарному плану, детализированному в плане обучения и создания документации для конечных пользователей.

Проведение обучения

·        Группа по обучению и документации отвечает за отбор персонала для принятия участия в обучении конечных пользователей, обеспечении необходимых инструментов и руководства, планировании занятий и присутствии персонала.

·        Члены группы внедрения должны присутствовать в организациях проведения обучения, чтобы обеспечивать экспертное руководство конечными пользователями как участниками процесса обучения. Кроме того, в обязанности членов группы внедрения входит отслеживание и разрешение вопросов, получаемых референтами во время учебных мероприятий.
