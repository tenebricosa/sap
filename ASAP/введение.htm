Введение в ASAP

Методология внедрения SAP R/3

Как и любая другая программная система управления бизнесом, SAP R/3 требует трудоемкого и  длительного процесса внедрения. Для упрощения и унификации этого процесса компанией SAP AG была разработана методология внедрения, состоящая из  последовательности шагов («Маршрутная карта»), предписывающих четко определенные этапы проекта внедрения, определенные цели проекта и каждого из этапов, набор завершающих документов по этапу и проекту, список решаемых задач; базы знаний по проекту; дополнительных сервисов. Эта методология охватывает все этапы внедрения от составления представления «как есть» до этапа ввода в эксплуатацию и проверка работоспособности системы.

Общие сведения об ASAP

AcceleratedSAP (ASAP) представляет собой поставляемое компанией SAP AG программное решение для внедрения. ASAP объединяет в своем составе несколько компонентов, которые работают совместно, обеспечивая быстрое и эффективное внедрение системы R/3.

На рисунке 1 представлены основные инструменты ASAP.

Рис. 1 – Инструменты ASAP

 

 

Основным инструментом ASAP, поддерживающим методологию внедрения системы, является «Ассистент по внедрению» (Implementation Assistent). Он представляет собой приложение, состоящее из следующих компонентов:

·        «Маршрутной карты»

·        Акселераторов (ускорители внедрения)

·        Вспомогательной документации («Уголок знаний»)

 

В маршрутной карте в виде дерева процессов и с высокой степенью подробности определяются требования к системе и процессу разработки и последовательность действий для скорейшего внедрения системы

 

Каждый акселератор представляет собой документ (ссылку на который можно найти в конце каждой ветви дерева маршрутной карты), ускоряющий работу по внедрению системы

Типы акселераторов:

·        Шаблоны

Это документы, содержащие таблицы или списки вопросов с пояснениями к заполнению. Например, шаблон для плана проведения стресс-теста.

·        Инструкции

Большинство из них являются инструкциями к заполению шаблонов или к выполнению задачи, предусмотренной в маршрутной карте.

«Уголок знаний» используется как хранилище вспомогательной информации, которая необходима для решения непредвиденных проблем в процессе осуществления внедрения системы. Документы в этом хранилище накоплены за многолетнюю практику внедрения с помощью ASAP. Любой участник проектной группы может дополнить «Уголок знаний» своим документом, в котором будет описана возникшая у него проблема и пути ее решения.

Вторым по важности модулем, является инструмент «База Данных Вопросов и Ответов» (Q&Adb, БД ВиО).

БД ВиО включает ссылочную модель SAP R/3 – это условное представление системы в виде деревовидной структуры, где, в конце каждой ветви находятся соответствующие опросники, шаблони, описания бизнесс-процессов по соответствующей бизнес-сфере предприятия.

Инструмент, содержит базу данных  проблем, возникающих при внедрении. Проблема заносится в БД ответственным лицом, вместе с требованиями по решению. С этой проблемой могут так же, ознакомится сотрудники, работающие в смежных областях, и для кого эта проблема является важной информацией.

БД ВиО это основной инструмент, используемый на фазе 2 для создания концептуального бизнес-проекта, основного списка бизнес-процессов, состава организационного списка – документы, играющие основополагающую роль в течение всего этапа внедрения системы SAP.

 

«Инструмент адмимнистрирования» (Admin Tool) служит для регистрации пользователей и присвоения им соответствующих прав, по работе с БД ВиО и Ассистентом по внедрению, т.е., профилей. Здесь так же можно увидеть все созданные проекты (в зависимости, от сферы деятельности подразделения предприятия) для внедрения.
